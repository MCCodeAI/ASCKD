Device Types

The device type of a device determines several characteristics of the device.

The device type is specified as an argument to the CreateDevice function when creating the device. The device types are enumerated in the DeviceType enumerator.

When the device type is not specified, a device of the Normal device type will be created. Most user applications should only use the Normal device type.

The four kinds of devices are listed in the following table.

Device Type Low Priority ExitWOCnt
Normal No No
DeviceTypeLowPriority Yes No
DeviceTypeExitWOCnt No Yes
DeviceTypeLowpriorityExitWOCnt Yes Yes

There is a dedicated API process thread for each normal (non-low priority) device. The command processing and response are very fast for normal devices. As long as a CPU core allocated to the real time operating system is available, API function calls are immediately processed and the response is returned. In the case of Function Calls That Are Blocking, the response may be delayed until the end of the next communication cycle.

For low priority devices, commands are processed one by one after the system process is finished at the end of each communication cycle. If there is a command already being processed, commands from other low priority devices must wait. Low priority devices are only used for tasks which do not require fast response times.

Whenever a device is closed with the CloseDevice function, the engine checks for the number of remaining devices. When there are no remaining devices, the engine will exit. Device types labeled "ExitWOCnt" are not counted when checking for the number of remaining devices. When the engine exits with "ExitWOCnt" devices remaining, API functions that are subsequently called through these devices will return the IMLibIsNotRunning error.
